### A1 配置优先级说明

- Spring Boot 配置优先级从高到低如下：

  1. 命令行参数
  2. 环境变量
  3. 外部配置文件
  4. 环境配置文件（如 application-dev.yml / application-prod.yml）
  5. 主配置文件（application.yml）

  示例说明：

  - 使用 `--server.port=9090` 启动时，会覆盖配置文件中的端口配置。
  - 使用环境变量 `DB_PASSWORD` 时，会覆盖 `application-prod.yml` 中的 `${DB_PASSWORD}` 占位符。
  - 当激活 `prod` 环境时，会优先加载 `application-prod.yml` 中对应配置。

## 五、问题与解决记录

### 1. 遇到的问题

- 问题 1：切换 `prod` 环境后，项目仍然以 `dev` 环境启动，导致访问 `8088` 端口时报连接失败。
- 问题 2：使用 Apifox 测试文件上传接口时，最开始使用了 `JSON` 请求体，导致接口返回 `Content-Type is not supported`。
- 问题 3：阿里云短信接口联调时出现 `Specified signature does not match our calculation`，短信无法正常发送。
- 问题 4：配置 Gmail 邮件发送功能时，接口返回 `Mail server connection failed`，无法连接到 `smtp.gmail.com:587`。
- 问题 5：调用 DeepSeek 接口后，虽然外部接口已经返回数据，但项目响应时报 `Type definition error: [simple type, class cn.hutool.json.JSONNull]`。

### 2. 解决方案

- 解决方案 1：检查 IDEA 运行配置后发现 `Active profiles` 被固定写成了 `dev`，将其改为 `prod` 或清空后重新启动，问题解决。
- 解决方案 2：文件上传接口要求使用 `multipart/form-data`，在 Apifox 中将 Body 改为 `form-data`，参数名设置为 `file` 并选择本地文件后，上传成功。
- 解决方案 3：重新核对阿里云 `AccessKeyId`、`AccessKeySecret`、签名名称和模板编码，同时清理环境变量中的多余空格，逐项排查验签失败原因。
- 解决方案 4：检查邮件环境变量、授权码和 SMTP 参数后，确认代码配置格式正确；对于 Gmail 连接超时的问题，优先判断为当前网络无法连通外部 SMTP 服务，可更换网络环境或改用更容易联调的邮箱服务继续测试。
- 解决方案 5：排查后发现返回值中直接放入了 Hutool 的 `JSONObject`，Spring Boot 在序列化 `JSONNull` 时发生异常，因此将原始响应改为普通字符串或基础类型，最终恢复接口正常返回。
